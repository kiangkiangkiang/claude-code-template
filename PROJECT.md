## 專案目標

這是一個 Claude Code 工作流程模板專案，目標是建立一套標準化的 AI 輔助開發流程。

### Epic 概念

1. **標準化 AI 開發流程**: 建立可重複使用的 Claude AI 開發協作模式
2. **文件驅動開發**: 透過結構化文件指導 AI 理解專案脈絡
3. **Git 流程整合**: 將任務管理與版本控制深度整合

## 核心價值

1. **可追溯性**: 所有開發決策和變更都有完整記錄
2. **可重現性**: 透過文件和工具，任何時候都能重現開發流程
3. **可擴展性**: 模板化的結構易於應用到其他專案
4. **協作效率**: 清晰的流程和文件減少溝通成本

## 開發守則

### 文件管理
- ✅ 所有系統級文件放在 `.claude/docs/`
- ✅ CLAUDE.md 只記錄宏觀概念，不放具體細節
- ✅ PROJECT.md（本文件）隨專案進展持續更新
- ✅ TASK.md 記錄當前任務，完成後清空

### Git 工作流程
- ✅ 每個任務拆分為多個 issue
- ✅ 每個 issue 對應一個開發分支
- ✅ 使用 `.claude/tools/git` 工具管理 Git 操作
- ✅ Commit 訊息要清晰描述變更內容
- ✅ PR 描述要完整說明解決的問題

### 任務執行
- ✅ 啟動時先查看 TASK.md 和 GitHub Issues
- ✅ 接收新任務時判斷是否與前次相關
- ✅ 任務完成時更新 PROJECT.md 記錄進度
- ✅ 使用 TodoWrite 工具追蹤任務進度
